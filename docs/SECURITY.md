# Security -- Big 5 Consulting MCP Server

## 보안 모델

이 프로젝트는 로컬 MCP 서버로 외부 네트워크 통신이 없습니다.
보안 범위는 로컬 파일 시스템 보호와 데이터 무결성에 집중합니다.

## 데이터 보호

### 저장 위치
```
~/.big5/
├── profiles/    # 개인 성격 프로필 (Big5 점수, 이름, 메타데이터)
└── surveys/     # 설문 진행 세션 (답변 기록)
```

### 보호 규칙
1. **git 제외**: `~/.big5/` 데이터는 `.gitignore`에 등록
2. **환경변수 보호**: `.env` 파일은 `.gitignore`에 등록
3. **사용자 삭제 권한**: `manage_profile` delete 액션으로 프로필 삭제 가능
4. **외부 전송 없음**: 모든 데이터가 로컬 파일 시스템에만 저장

### 민감 정보
| 데이터 | 민감도 | 보호 방식 |
|--------|--------|----------|
| Big5 점수 | 중간 | 로컬 저장, 외부 전송 없음 |
| 이름 | 중간 | 로컬 저장 |
| 메타데이터 (나이, 직업 등) | 높음 | 로컬 저장, 선택적 입력 |
| 설문 응답 | 낮음 | 완료 시 세션 삭제, 점수만 보존 |

## 파일 시스템 접근

### 접근 범위 제한
- **읽기**: `~/.big5/` 하위만 허용
- **쓰기**: `~/.big5/profiles/`, `~/.big5/surveys/` 만 허용
- **삭제**: 개별 프로필/세션 파일 단위만 허용
- 상위 디렉토리 탈출(path traversal) 방지: StorageManager에서 경로를 `path.join()`으로 구성

### 디렉토리 생성
- `StorageManager.initialize()`에서 `{ recursive: true }` 옵션으로 생성
- 디렉토리 생성 실패 시 서버 시작 실패 (process.exit)

## 인증/인가

### 현재 상태
- **인증 없음**: MCP 서버는 stdio 트랜스포트를 사용하여 별도 인증 불필요
- **인가 없음**: 단일 사용자 로컬 환경으로 권한 분리 불필요
- Claude Desktop이 프로세스를 직접 실행하므로 OS 레벨 접근 제어에 위임

### 향후 고려 사항
- HTTP 트랜스포트 전환 시 API 키 인증 필요
- 다중 사용자 환경 시 프로필별 접근 제어 필요

## 입력 검증

### MCP 스키마 레벨
- JSON Schema를 통한 입력 구조 검증
- enum 타입으로 허용 값 제한 (action, analysis_type 등)
- 숫자 범위 검증 (Big5 점수: 0-100, 응답: 1-5)
- 배열 크기 검증 (답변 배치: minItems=5, maxItems=5)

### 도구 레벨
- 필수 파라미터 존재 확인
- 프로필/세션 존재 확인 (null 검사)
- 세션 상태 전이 검증 (답변 완료 여부)
