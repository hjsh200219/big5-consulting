# Reliability Standards

> Big 5 Consulting MCP Server -- 신뢰성 기준 정의

## 서비스 특성

이 프로젝트는 MCP 서버로, Claude Desktop에서 로컬 프로세스로 실행됩니다.
웹 서비스와 달리 단일 사용자 환경이며, stdio 트랜스포트를 사용합니다.

## 데이터 무결성

### 저장소 보호 규칙
- **프로필 데이터**: `~/.big5/profiles/` 디렉토리의 JSON 파일
- **세션 데이터**: `~/.big5/surveys/` 디렉토리의 JSON 파일
- 파일 쓰기는 항상 전체 JSON을 덮어쓰기 (부분 업데이트 없음)
- 캐시와 파일 시스템 간 일관성 유지 (쓰기 시 캐시 동시 갱신)

### 데이터 유효성
- Big 5 점수 범위: 0-100 (정수)
- 설문 응답 범위: 1-5 (정수)
- 세션 상태 전이: start -> submit(xN) -> complete
- 완료된 세션은 삭제되고 프로필로 변환

## 에러 처리 기준

### 복구 가능 에러
| 에러 | 처리 방식 |
|------|----------|
| 프로필 미존재 | null 반환, 호출자가 에러 메시지 생성 |
| 세션 미존재 | null 반환, 호출자가 에러 메시지 생성 |
| 파일 읽기 실패 (ENOENT) | 정상 동작으로 처리 (null 반환) |
| 파일 삭제 실패 (ENOENT) | 성공으로 처리 (이미 없음) |

### 복구 불가 에러
| 에러 | 처리 방식 |
|------|----------|
| 디렉토리 생성 실패 | 서버 시작 실패 (process.exit) |
| 알 수 없는 도구 호출 | MCP 에러 응답 반환 |
| 파일 권한 에러 | 에러 로깅 후 null/false 반환 |
