# Core Beliefs -- 에이전트 우선 운영 원칙

## 원칙 1: 문서가 코드베이스의 진실의 원천

에이전트는 코드를 읽기 전에 문서를 먼저 읽습니다.
따라서 문서가 항상 코드의 현재 상태를 정확히 반영해야 합니다.

- `agent.md` -> 프로젝트 개요, 핵심 규칙, 빌드 명령
- `ARCHITECTURE.md` -> 도메인 맵, 레이어 구조, 의존성 방향
- `docs/` -> 상세 규칙, 품질 기준, 보안 규칙

## 원칙 2: 아키텍처 규칙은 불변

레이어 의존성 규칙은 모든 코드 변경에서 반드시 준수해야 합니다.

- **Import 방향**: 상위 -> 하위만 허용
- **도구 독립성**: tools/ 파일 간 직접 import 금지
- **타입 리프**: types/index.ts는 내부 모듈을 import하지 않음
- **단일 저장소**: StorageManager가 유일한 데이터 접근점

이 규칙을 위반하는 변경은 어떤 이유로도 허용하지 않습니다.
상세 규칙: [layer-rules.md](layer-rules.md)

## 원칙 3: 빌드 성공이 최소 품질 게이트

모든 코드 변경 후 `npm run build` 성공을 확인합니다.
TypeScript strict 모드에서 컴파일 에러가 없어야 합니다.

## 원칙 4: 점진적 개선

큰 리팩토링보다 작은 개선을 반복합니다.

- 기술 부채는 [tech-debt-tracker.md](../exec-plans/tech-debt-tracker.md)에서 추적
- 우선순위(P0-P3)에 따라 점진적으로 해결
- 품질 등급은 [QUALITY_SCORE.md](../QUALITY_SCORE.md)에서 추적

## 원칙 5: 데이터 프라이버시 우선

- 모든 사용자 데이터는 로컬 (`~/.big5/`)에만 저장
- 외부 API 호출이나 데이터 전송 없음
- .env, 테스트 데이터, 사용자 프로필은 git에 포함하지 않음

## 원칙 6: 에이전트 자율성과 일관성

에이전트는 이 문서 체계를 통해 일관된 판단을 내립니다:
1. `agent.md`에서 프로젝트 맥락 파악
2. `AGENTS.md`에서 작업 유형별 체크리스트 확인
3. 관련 docs/ 문서에서 상세 규칙 확인
4. 코드 변경 실행
5. `npm run build`로 검증
